# Руководство пользователя

## 1. Введение

Мобильное приложение «Диспетчер на колёсах» предназначено для автоматизации управления автопарком. Приложение поддерживает три роли пользователей: **Клиент**, **Диспетчер** и **Водитель**. В зависимости от роли, пользователю доступны разные функции.

---

## 2. Установка приложения

### 2.1. Установка через APK

1. Скачайте APK файл
2. На телефоне разрешите установку из неизвестных источников
3. Откройте APK и нажмите «Установить»

### 2.2. Установка через отладку

```bash
adb install app-debug.apk
```

---

## 3. Регистрация и вход

### 3.1. Регистрация

1. Запустите приложение
2. Нажмите **«Нет аккаунта? Зарегистрируйтесь»**
3. Выберите роль:
   - **Клиент** — управление автопарком
   - **Диспетчер** — обработка событий
   - **Водитель** — получение команд
4. Заполните поля: ФИО, телефон, email, пароль
5. Для диспетчера и водителя выберите автопарк из списка
6. Нажмите **«Зарегистрироваться»**

### 3.2. Вход

1. Введите email и пароль
2. Нажмите **«Войти»**

### 3.3. Выход

Нажмите кнопку **«Выйти»** в правом верхнем углу главного экрана.

---

## 4. Роль: Клиент

### 4.1. Панель управления

После входа открывается главный экран с:
- Количеством транспортных средств
- Количеством отчётов
- Разделами: Транспорт, Водители, Отчёты

### 4.2. Управление транспортными средствами

**Просмотр списка:** откройте раздел «Транспортные средства»

**Добавление ТС:** нажмите кнопку «+» → заполните госномер и модель → «Сохранить»

**Редактирование:** нажмите ✏️ → измените данные → «Сохранить»

**Удаление:** нажмите 🗑️ → подтвердите удаление

### 4.3. Управление водителями

**Просмотр списка:** откройте раздел «Водители»

**Привязка к ТС:** нажмите «Привязать ТС» → выберите транспорт → «Сохранить»

**Отвязка:** нажмите «Отвязать» → подтвердите действие

### 4.4. Отчёты

**Просмотр:** откройте раздел «Отчёты»

**Скачивание:** нажмите на карточку отчёта → файл скачается на устройство

---

## 5. Роль: Диспетчер

### 5.1. Список событий

После входа открывается список событий с информацией:
- Тип события (падение топлива, превышение скорости, длительный простой)
- Приоритет (критический, высокий, средний, низкий)
- Краткое описание
- Время возникновения

**Цветовая индикация приоритетов:**
- Красный — критический
- Оранжевый — высокий
- Жёлтый — средний
- Зелёный — низкий

### 5.2. Фильтрация событий

1. Нажмите кнопку **«Фильтр»**
2. Выберите параметры: тип события, приоритет, статус
3. Нажмите **«Применить»**

Активные фильтры подсвечиваются синим цветом. Для сброса нажмите **«Сбросить»**.

### 5.3. Детали события

Нажмите на карточку события для просмотра:
- Полного описания
- GPS-координат
- Текущего статуса

### 5.4. Обработка события

На экране деталей доступны действия:
- **«Взять в работу»** — изменить статус на «В обработке»
- **«Закрыть»** — завершить обработку события

### 5.5. Отправка команды водителю

1. На экране деталей события введите текст команды
2. Нажмите **«Отправить команду»**
3. Водитель получит push-уведомление

---

## 6. Роль: Водитель

### 6.1. Список команд

После входа открывается список полученных команд с информацией:
- Текст команды
- Статус (отправлено, доставлено, прочитано, подтверждено, ошибка)
- Время отправки

**Цветовая индикация статусов:**
- Оранжевый — отправлено
- Синий — доставлено
- Зелёный — прочитано
- Тёмно-зелёный — подтверждено
- Красный — ошибка

### 6.2. Детали команды

Нажмите на карточку команды для просмотра:
- Полного текста команды
- Времени отправки и доставки
- Текста ошибки (если есть)

### 6.3. Подтверждение команды

1. На экране деталей команды введите текст подтверждения
2. Нажмите **«Подтвердить выполнение»**
3. Статус команды изменится на «Подтверждено»

---

## 7. Push-уведомления

При получении новой команды водитель получает push-уведомление:
- Заголовок: «📢 Новая команда от диспетчера»
- Текст: текст команды (сокращённо)

При нажатии на уведомление открывается экран деталей команды.

---

## 8. Возможные ошибки и их решение

| Ошибка | Решение |
|--------|---------|
| Не удаётся войти | Проверьте email и пароль. Нажмите «Забыли пароль?» для восстановления |
| Не загружаются события | Проверьте подключение к интернету. Потяните экран вниз для обновления |
| Не отправляется команда | Проверьте, выбран ли водитель и есть ли у него привязка к ТС |
| Не приходят push-уведомления | Проверьте настройки уведомлений в телефоне. Разрешите уведомления для приложения |

## 9. FAQ

**Вопрос:** Как сменить роль?  
**Ответ:** Роль назначается при регистрации и не может быть изменена. Создайте новый аккаунт с нужной ролью.

**Вопрос:** Как добавить нового водителя?  
**Ответ:** Войдите под ролью «Клиент», откройте раздел «Водители», зарегистрируйте водителя с ролью «Водитель».

**Вопрос:** Как привязать водителя к машине?  
**Ответ:** Войдите под ролью «Клиент», откройте «Водители», нажмите «Привязать ТС» у нужного водителя, выберите транспорт.

**Вопрос:** Почему диспетчер не видит события?  
**Ответ:** Убедитесь, что диспетчер привязан к автопарку (клиенту) и транспортные средства также привязаны к этому клиенту.

**Вопрос:** Как узнать IP для подключения по Wi-Fi?  
**Ответ:** Выполните в терминале `ipconfig` (Windows) или `ifconfig` (Mac/Linux), найдите IPv4-адрес.
